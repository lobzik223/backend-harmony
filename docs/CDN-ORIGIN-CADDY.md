# Ориджин за CDN: Caddy только по HTTP на порту 80

Чтобы CDN (CDNvideo) мог обращаться к вашему серверу **147.45.243.157:80** и получать ответ (а не редирект 308), Caddy на ориджине должен **отдавать контент по HTTP на порту 80**, без редиректа на HTTPS.

Иначе при запросе к `http://147.45.243.157:80` CDN получает 308 на `https://147.45.243.157/`, а при обращении по HTTPS к IP сервер выдаёт ошибку TLS (сертификат выдан на имя домена, а не на IP).

---

## Что поменять в Caddyfile на сервере

Нужно, чтобы по **порту 80** запросы обрабатывались по заголовку **Host** и отдавался контент (без редиректа на HTTPS).

**Вариант 1: ориджин только HTTP (порт 80)**

Если весь трафик к сайту идёт через CDN, а на 147.45.243.157 заходят только узлы CDN, можно оставить на ориджине только HTTP:

```caddyfile
# Ориджин для CDN — только порт 80, без редиректа на HTTPS
http://api.harmonymeditation.online {
    reverse_proxy 127.0.0.1:3000
}

http://panel.harmonymeditation.online {
    root * /root/panel-harmony/dist
    try_file {path} /index.html
    file_server
}

http://harmonymeditation.online {
    redir https://panel.harmonymeditation.online{uri} permanent
}
```

Тогда CDN обращается к `http://147.45.243.157` с заголовком `Host: api.harmonymeditation.online` (или panel) и получает ответ 200 с телом, а не 308.

**Вариант 2: и HTTP для CDN, и HTTPS для прямого доступа**

Если нужен и доступ по HTTPS напрямую к домену (например, для продления сертификатов), оставьте текущие блоки с `https://...`, но добавьте **отдельные** блоки для HTTP, которые отдают контент, а не редирект. В Caddy при наличии и `http://`, и `https://` для одного имени поведение может зависеть от версии; надёжный вариант — на ориджине оставить только HTTP (вариант 1), а сертификаты продлевать, например, по DNS-01 или с другого места.

---

## После правок

```bash
sudo caddy validate --config /etc/caddy/Caddyfile
sudo systemctl reload caddy
```

Проверка с другой машины (как у CDN):

```bash
curl -i -H "Host: api.harmonymeditation.online" http://147.45.243.157/
```

Ожидается ответ 200 (или от бэкенда), без `Location: https://...`.
